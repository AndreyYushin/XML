GIT Homework 1 part 2

Для выполнения задания у вас должен быть установлен для Windows - GitBash.
Создан аккаунт в GitHub

Все шаги сценария выполняйте в терминале GitBush, Terminal, в папке под гитом.


Как отправить ДЗ на проверку.
 1. Создайте текстоовый файл как в первом ДЗ по Terminal.
 2. Сценарий перенесите в этот файл.
 3. На против каждого действия - напишите команду в GitBash
XML
 21. Создать внешний репозиторий c названием XML - заходим в GitHub, выбираем создать New Repositories, в конструкции выбираем сделать публичным, добавить файл READMI.
 22. Клонировать репозиторий XML на локальный компьютер - в нашем новом репозитории справа вверху нажимаем кнопку Code, копируем HTTPS код, заходим в Terminal, создаем директорию, в ней пишем команду git clone и подставляем нашу HTTPS ссылку.
 23. Внутри локального XML создать файл “new.xml” - touch new.xml
 24. Добавить файл под гит - git add new.xml
 25. Закоммитить файл - git commit -m "List"
 26. Отправить файл на внешний GitHub репозиторий - git push
 27. Отредактировать содержание файла “new.xml” - написать информацию о себе (ФИО, возраст, количество домашних животных, будущая желаемая зарплата). Всё написать в формате XML - выполнено.
 28. Отправить изменения на внешний репозиторий - git add new.xml, git status, git commit - m, git push
 29. Создать файл preferences.xml - vim preferences.xml
 30. В файл preferences.xml добавить информацию о своих предпочтениях (Любимый фильм, любимый сериал, любимая еда, любимое время года, сторона которую хотели бы посетить) в формате XML - выполнено.
 31. Создать файл sklls.xml добавить информацию о скиллах которые будут изучены на курсе в формате XML - vim skills.xml выполнено.
 32. Сделать коммит в одну строку - git commit -m
 33. Отправить сразу 2 файла на внешний репозиторий - git add . , git commit нажимаем enter пишем комментарий к каждому, нажимаем esc вводим :wq, git push
 34. На веб интерфейсе создать файл bug_report.xml - внутри нашего XML нажимаем кнопку Add file ниже выбираем Create new file, прописываем ему имя и формат, нажимаем Commit changes, в появившемся окне выбираем Commit directly to the main branch и нажимаем Commit changes.
 35. Сделать Commit changes (сохранить) изменения на веб интерфейсе - описанно в пункте 34
 36. На веб интерфейсе модифицировать файл bug_report.xml, добавить баг репорт в формате XML - открываем свой файл, справа нажимаем кнопку Edit this file
 37. Сделать Commit changes (сохранить) изменения на веб интерфейсе - описанно в пункте 34
 38. Синхронизировать внешний и локальный репозиторий XML - заходим в Terminal и выполняем команду git pull
